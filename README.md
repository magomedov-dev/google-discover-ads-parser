# GoogleAdsParser

Парсер рекламы из ленты **Google Discover** (приложение Google на Android).
Листает ленту на реальном устройстве, находит карточки с пометкой «Sponsored»,
снимает их и собирает по каждой рекламе скрин, медиа, текст, канал и ссылки.

Построен на [`axonctl`](https://github.com/magomedov-dev/axonctl) — управление
устройствами через accessibility/adb.

## Возможности

- 🔎 Поиск рекламы в ленте Discover и определение её формы: **video / carousel / image**.
- 🖼️ Скрины: общий скрин карточки и отдельный скрин медиа (с выравниванием низа
  картинки к рабочей области, если она не влезает целиком).
- 🏷️ Извлечение **заголовка** и **канала** рекламодателя.
- 🔗 Сбор ссылок:
  - полный URL лендинга (через page-info Chrome);
  - google-style share-ссылка (через «Share link»);
  - ссылка на видео (через «Поделиться» / «Копировать ссылку» в плеере, с чтением
    буфера обмена там, где прямого поля нет).
- 🧹 Блок оценки рекламы уводится ниже рабочей области, чтобы не попасть в скрин.
- ♾️ Бесконечный сбор: листание → обновление ленты → перезапуск приложения.
- 🛡️ Устойчивость: адаптивные ожидания под медленный интернет, изоляция сбоев по
  итерациям, повторный запуск, обработка случайно открывшегося Chrome.
- 🧾 Дедуп по «канал + заголовок», сквозная нумерация (переживает перезапуск).
- 🪪 Уникальное логирование ошибок: на каждую ошибку — папка с трейсбеком и снимком UI.

## Требования

- Python ≥ 3.13, [uv](https://docs.astral.sh/uv/)
- Android-устройство с включённой отладкой по USB и `adb` в `PATH`

## Установка

```bash
uv sync
```

## Запуск

```bash
uv run main.py                       # все подключённые устройства, config.toml
uv run main.py --device 276bcca9     # конкретное устройство
uv run main.py --swipes 40 --concurrency 4
uv run main.py --config my.toml -o ads2
uv run main.py --list-devices        # показать подключённые устройства
```

Устройства определяются по приоритету: `--device` → секция `[fleet]` в конфиге →
автоопределение через `adb devices`. Отдельный `fleet.toml` не нужен.

## Конфигурация (`config.toml`)

Все параметры опциональны; флаги CLI переопределяют значения из файла.

```toml
[scrape]
swipes = 25            # свайпов за одну обработку ленты
output_dir = "ads"     # каталог собранной рекламы
error_dir = "errors"   # каталог логов ошибок
ready_timeout = 15.0   # ожидание элементов (медленный инет), с
ad_load_timeout = 3.0  # ожидание загрузки сайта/диалогов, с
ad_dwell_min = 2.0     # «просмотр» сайта рекламы, с
ad_dwell_max = 5.0
# ... остальные тайминги см. в config.toml

[scrape.device_names]
# "276bcca9" = "phone1"   # имя подкаталога для устройства (иначе — по серийнику)

[fleet]
concurrency = 8
devices = []           # пусто — автоопределение через adb
```

## Что сохраняется

```
ads/<устройство>/adNNN/
├── general.png      # общий скрин карточки (без блока оценки)
├── media.png        # скрин картинки/видео/карусели
├── structure.json   # UI-структура карточки
├── ui.html          # интерактивный снимок экрана
└── meta.json        # форма, текст, канал, page_url, landing_url, video_url
```


## Как это работает

1. Запуск приложения Google, ожидание ленты.
2. Определение рабочей области (между строкой поиска и низом ленты).
3. Листание медленными свайпами, поиск карточек «Sponsored».
4. Обработка рекламы: выравнивание, скрины, текст/канал, открытие и сбор ссылок
   (Google Play пропускается).
5. После прохода — обновление ленты, полное закрытие и перезапуск приложения.
